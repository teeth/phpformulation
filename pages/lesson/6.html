We have finished writing the code to the PHP scripted web form. The final thing to do now is to
make the file, which is currently a HTML file, a PHP file. This is simple to do. Just Save As with a
“.php” extension to the end of the filename. If you’re using Notepad, make sure the Save as type is
set to All Files. You may be using another text editor, which can simply save the file as a PHP file
using the Save as type drop down list. So in that case a “.php” extension doesn’t need to be added.

Now the file is a PHP file, refresh your browser and you’ll likely get a page not found error. That’s
because you can’t simply open a PHP file using a web browser. You have to upload it to a server. The
server can be a web host (make sure your web host supports PHP) or you can run your computer as
a server. In the example of the web form being sent (on the next page), I have uploaded the web

form to a web host. In regards to running your computer as server, there are two suggestions I will make:
  • You can install the latest version of PHP onto your computer, from the PHP website. Then
    you have to configure it to run with Apache. The latest version of Apache can be
    downloaded on their website.
  • Another option is to install XAMPP onto your computer. This can be downloaded from their
    website, via Sourceforge.
Out of the two options, the latter is the one I would advise users to choose. Reason being is that
XAMPP installs Apache and PHP for you, along with other programs that may be useful to you.
Additional information, if you’re running your computer as a server you have to make sure you have
your PHP.ini file configured. Check the PHP website for more information.
